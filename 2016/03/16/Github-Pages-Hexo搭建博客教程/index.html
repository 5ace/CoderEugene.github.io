Github Pages+Hexo搭建博客教程

发表于 2016-03-16   |   分类于 杂文   |  


我已经通过Github Pages和Hexo搭建了个人blog，作为开篇，这里也简单的讲述一下搭建过程，如果能帮助更多人搭建blog,那是再好不过了，那么，现在开始了！

搭建安装软件

搭建过程中会涉及到git的一些基本操作，具体会在下文讲到，
如果你需要详细了解git操作，推荐Git教程–廖雪峰官方教程，内容简洁生动，易学易用

  • Node.js
  • git

使用GitHub Pages

当然，首先你得先创建一个github账号，然后创建一个「New Repository」

创建完毕后，可以点击Repository中的settings来检验是否已创建成功

配置SSH

关于配置SSH KEY，在这篇文章中，我不详细介绍，Git教程–廖雪峰官方教程 中会有很好的介绍和教程

安装Hexo

在桌面上右键打开git bash

安装完毕后，在本地创建一个Hexo文件夹，在该文件中打开git bush

输入完毕后，Hexo会自动在目标文件夹建立网站所需要的所有文件

增加Hexo主题

hexo-theme-next 是我现在使用的主题

在Hexo目录下打开git bush

克隆完主题后，就有启动主题，在Hexo中修改 _config.yml，将theme改成next,
这个时候就在本地搭建好博客了，那么，检验一下是否真的已搭建好了

运行完毕后，输入后访问http://localhost:4000/，如果出现blog则代表本地搭建成功

绑定域名

  • 打开github，在Repository的根目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如blog.zzcoder.cn
  • DNS解析：将你的域名解析用类型CNAME解析即可

结语

到此为止，你的个人博客应该已经配置完成了，但是后续对于主题的一些细节设置还需要你细细斟酌，这里也不再叙述了，而我也在最后介绍下一些常用操作

  • 写文章：输入以下指令后，在Hexo\source_posts中会自动创建一个.md文件用来编辑你的文档

  • 插入图片：

    • 如果要插入图片，为了图片不会太大而影响美观，我们先需要配置一下，这个在官方介绍已有介绍

      如何更改内容区域的宽度？

      NexT 对于内容的宽度的设定如下：

      • 700px，当屏幕宽度 < 1600px

      • 900px，当屏幕宽度 >= 1600px

      • 移动设备下，宽度自适应

      如果你需要修改内容的宽度，同样需要编辑样式文件。 编辑主题的 source/css/_variables/custom.styl 文件，新增变量：

    • 同时为了提高网站打开速率，我们需要一个图床，这里使用七牛,具体使用步骤:
