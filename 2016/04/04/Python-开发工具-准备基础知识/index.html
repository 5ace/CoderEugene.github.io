Python-开发工具+准备基础知识

发表于 2016-04-04   |   分类于 Python   |  


虽然至今我最熟悉的语言还是 C++ ，但是 C++ 写一些算法还是挺麻烦的（特别是涉及到ADT，当然条件是不能用STL，哈哈），所以我决定花个几天时间来学习Python，这是一门非常优秀的高级语言，易学易用，非常适合写算法，因为它不会总让我的思绪停留在语言层面上，而是更注重算法本身。同时呢，对于网络数据的抓取以及小应用的开发也离不开它，由于我已经接触过类似C++，Java等语言，所以对于基本的知识我都会一笔带过，大部分精力集中在Python本身的新特性上，那么，接下来是我的Python第一课

选择Python工具

其实最开始我写Python是用官方的python解释器)的，这确实还不错，并且很适合初学者，使用非常简单和方便，但是，由于我用多了类似VS，Eclipse等IDE，对于没有智能补全等功能实在不能忍，所以现在使用的是Python的IDE中比较优秀的PyCharm,用起来非常顺手，并且它还能写HTML，所以如果你也想要更高效的码字，我推荐PyCharm

选择Python版本

如果你现在是刚开始学Python，也许你会发现Python是有2个版本的，也就是Python2.x和Python3.x，由于3.x版本相对2.x版本来说改动较大，所以很多支持2.x的模块或者代码都不适用于3.x，因此需要谨慎的选择版本，3.x虽然是最新版，但是由于网上很多模块或者解决问题都是针对2.x而不能同样适用在3.x中，所以你也许会在学习中花更多时间。不过，Python3.x是未来的趋势，如果你是刚开始学Python，我推荐你学习Python3.x，善于探索并解决问题本身也是程序猿最基本的素养，并且现在越来越多的模块支持Python3.x了

理解Python3字符编码

提及编码前，我们要先了解几个必须了解的知识

Unicode:编码符号集，它规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储

UTF-8:一种Unicode的实现方式

ASCII:使用指定的7位或8位二进制数组合来表示128或256种可能的字符

1. 这三种字符编码都是将字符对应不同的二进制字节，其中ASCII编码是1个字节，Unicode编码通常是2个字节（可以看作中间连接层，它不能作为存储编码，需要类似UTF-8等实现编码来存储），而UTF-8编码常用的英文字母被编码成1个字节，汉字通常是3个字节
2. 在计算机内存中，统一使用Unicode编码，当需要保存到硬盘或者需要传输的时候，就转换为UTF-8编码。也就是说，打开一个UTF-8编码的文件时，需要先转换成Unicode放在内存中，然后当要保存文件时，将Unicode转为UTF-8保存，ASCII是同样的道理

输入和输出

  • 输入
    • input([str])函数：括号内为可选字符串类型参数，用于提示输入内容

  • 输出

    • print()函数，括号内可传入多个字符串，中间用逗号隔开，并且1个逗号输出一个空格

      • 由于在print()函数默认在输出后换行，所以如果你希望输入后不换行，可以在参数中添加end=””，end=”” 表示打印对象以什么结尾

      • 在print()函数中添加一对三引号： """ 或 '''可以按照内容的格式输出
