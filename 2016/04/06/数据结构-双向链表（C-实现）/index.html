为什么在上述代码中添加了友元函数？

因为，嵌套类中内部类和外围类互相没有特权，外部类与内部类其实联系并不大，外部类仅仅限定了内部 类类名的作用域范围。（嵌套类可以访问外围类的静态成员变量，即使它的访问权限是私有的），而c++中的友元类可以访问包括私有成员的各种成员，所以，为了list能通过其对象访问其私有成员，添加了友元函数

如何判断在继承中什么需要重写？

我们需要知道以下几点

  1. c++继承中会出现同名覆盖
  2. 基类重载的运算符在继承中若是该运算符未改变（包括参数，返回值等），是可以继承的，但是若是改变了任意一项，都需在子类重写，而且它会屏蔽基类同名方法（而且是会屏蔽同名基类的所有同名重载函数）
  3. 因为const也是重载的一个标志，即有无const（访问函数和修改函数）代表重载该函数，所以在子类中重载其中任意一个都会把基类的无论有无 const函数都屏蔽，一般情况下是要在子类再写一遍另一个所需要的函数，即使内容完全相同。
  4. 容易令人忽视的是，如果在基类的函数无const，而子类的同名函数含有const同样会屏蔽基类所有同名函数。
  5. 友元函数不能继承

为什么子类需要显式继承基类构造函数

构造原则如下:

  1. 如果子类没有定义构造方法，则调用父类的无参数的构造方法
  2. 如果子类定义了构造方法，不论是无参数还是带参数，在创建子类的对象的时候,首先执行父类无参数的构造方法，然后执行自己的构造方法
  3. 在创建子类对象时候，如果子类的构造函数没有显示调用父类的构造函数，则会调用父类的默认无参构造函数
  4. 在创建子类对象时候，如果子类的构造函数没有显示调用父类的构造函数且父类自己提供了无参构造函数，则会调用父类自己的无参构造函数
  5. 在创建子类对象时候，如果子类的构造函数没有显示调用父类的构造函数且父类只定义了自己的有参构造函数，则会出错（如果父类只有有参数的构造方法，则子类必须显示调用此带参构造方法）。

实现操作简述

成员函数主要分为对迭代器的操作和对对象的操作

  • 对迭代器的操作主要通过iterator的带参构造函数来实现，而插入和删除中由于友元类，list可以访问iterator的私有成员，所以可以直接通过iterator对象来调用它的私有成员Node指针current从而实现插入和删除

类模板注意事项

  1. 模板类的声明和实现最好全放在同一个.h文件中，否则会出现编译不通过，另一个解决办法是在main函数中include “.cpp”文件
  2. 模板类的实现中类型参数要加typename（例如返回值类型，参数类型）(用于嵌套类)
  3. 模板类在实现中（特别是泛型类型时），注意模板参数全都得写，而且是左值，不是右值
    • 例如 template < typename T, typename Comparable = less< T > >定义的类A，实现函数时，应写成A< T,Comparable >,而不要错写成 A< T,less< T > >

结语

虽然双向链表是一个比较简单的数据结构，但是用C ++ 实现起来并不是那么简单，它会涉及到大量的C ++ 的语法层面，包括引用，继承，友元，类模板等知识，不过当我们写完这个数据结构会发现自己的C++水平又提高了不少，很多平时容易遗忘的知识都能重新拾起，那么，在写下次的数据结构的时候也会更轻松一些。另外，虽然我们花了这么久实现list，然而C ++ 标准库中的list早已包含了这些甚至更多更优秀的功能，我们可以很轻易的通过几行代码实现，但是当我们学会了list底层的实现，以后在调用或者重新造轮子添加新功能的时候，So easy！
