Python-验证码识别

发表于 2016-05-07   |   分类于 Python   |  


在以往的模拟提交中，当我遇到验证码时，采取的方式往往是下载验证码图片，然后人工识别输入。虽然这样处理是最好的方式了，但是为了程序的自动化，我们就需要通过代码对验证码进行。而验证码识别是一件比较困难的事情，同时，网上的验证码为了防止被破解一般会增加噪点，干扰线，变形等方式，这些就是我们现在要破解的

开发环境

  • Python3.4
  • Pillow (由于PIL不支持Python3.x，所以使用其衍生版)
  • tesseract-ocr

Pillow和tesseract-ocr的安装

  • Pillow直接使用Pycharm安装就可以了
  • tesseract的安装在github上都有写着，不过在windows下的安装略显复杂，好在官方文档有windows集成安装版的链接 tesseract-ocr-setup-3.05.00dev.exe tesseract-core-20150916.exe, 这两个版本都是是最新的，安装比较简单，不过我在安装的时候遇到无法识别问题，所以我的策略是两个都安装，并将tesseract-ocr-setup-3.05.00dev的tessdata文件夹提取出来并替换tesseract-core-20150916原有的
    • 检测tesseract的方式： 在cmd中输入tesseract（如果没有配置环境变量，可能还需要配置环境变量），如果正常输出信息则代表安装成功
    • 检验正确识别验证码格式： tesseract 带识别验证码名 识别出来的txt文件 例： tesseract 1.png text [如果这里无法识别png文件，你也可以尝试我的方法]
    • 设置tesseract只识别数字： 在tessdata/configs/digits目录下修改内容为 tessedit_char_whitelist 0123456789 即可（这个就是白名单，想识别谁就写进去），然后再cmd中识别验证码时增加一个参数 digits 例 tesseract 1.png text digits

验证码弱点

在验证码识别前我们需要堆验证码进行图像处理，这里我的处理主要有二值化，去噪点，去干扰线，字符切割（不过我在切割完后发现会出现无法识别的问题），它们的算法在常见验证码的弱点与验证码识别中讲的非常详细，我这里简单的归纳一下

  • 二值化：彩色验证码的背景和文字颜色的RGB是存在一定的阈值的，所以我们只需要遍历所有像素点，并将低于该阈值的点都变白，高于阈值的像素点变黑实现二值化
  • 去干扰线：干扰线一般情况下比文字细，所以我们对每个像素点的八个方向的像素点进行判断，如果周围有超过6个点的色差都比较大，说明这个点是噪点，以此类推，我们可以去除干扰线（但是对于干扰线和文字接近的情况这个算法不适用）
  • 字符切割：对于没有粘连的字符，每个字符之间会有整行或整列的白像素点（经过二值化后），所以我们只需要先垂直遍历，找到分界点，然后再水平切割即可
